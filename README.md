# portal-index-pages

一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为 HTML 文件的集合与发布点。

## 项目简介

`portal-index-pages` 旨在提供一个结构化的空间，用于存放、管理和展示各类独立的 HTML 页面。  
这些页面可以是信息索引、工具界面、静态展示页或其他任何基于 HTML 的内容。  
仓库本身不包含后端逻辑或动态数据，所有内容均为纯前端静态页面。

## 目录说明

仓库的根目录下按以下方式组织：

```
portal-index-pages/
├── pages/          # 存放所有独立 HTML 页面文件（或子目录）
├── assets/         # 共享的静态资源（CSS、JS、图片等）
├── index.html      # 仓库首页（可选，用于列出或导航到各页面）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件代表一个独立的 HTML 页面。建议按功能或主题命名。
- `assets/`：存放多个页面可能共用的样式、脚本或图片资源，避免重复。
- `index.html`：可作为入口页面，提供到各个页面的跳转列表。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，可单独部署或通过 GitHub Pages 访问。
- 页面之间原则上不互相依赖，除非显式引用共享资源。
- 每个页面应保持自包含或仅依赖 `assets/` 中的资源。
- 归档时请确保文件名清晰，避免特殊字符或空格。

## 维护说明

- 欢迎提交新页面或改进现有页面，请遵循仓库结构。
- 提交前请确保 HTML 文件可正常渲染，无外部无效链接。
- 不建议在页面中嵌入任何形式的追踪、广告或营销内容。
- 如有大文件或敏感信息，请勿提交至仓库。
